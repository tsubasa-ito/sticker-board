## 1. StoreKit Testing 環境セットアップ

### 1-1. Xcode での設定

1. `StickerBoard.xcodeproj` を開く
2. **Product > Scheme > Edit Scheme...** を選択
3. **Run > Options** タブを開く
4. **StoreKit Configuration** で `Products.storekit` を選択
5. これでシミュレータ・実機の両方で StoreKit Testing が有効になる

### 1-2. StoreKit Configuration ファイル

`Products.storekit` に以下の商品が定義済み:

| 商品ID | 種別 | 価格 | 期間 |
|--------|------|------|------|
| `com.solodev.StickerBoard.pro.monthly` | 自動更新サブスク | ¥380/月 | 1ヶ月 |
| `com.solodev.StickerBoard.pro.yearly` | 自動更新サブスク | ¥2,900/年 | 1年 |

### 1-3. StoreKit Testing の特徴

- **即時トランザクション**: 購入ダイアログが簡易表示され、すぐに完了する
- **サーバー不要**: ローカルで完結するため App Store Connect の設定不要
- **トランザクション管理**: Xcode の **Debug > StoreKit > Manage Transactions** で確認・操作可能

## 2. 手動確認チェックリスト

### 2-1. 無料プランの制限確認

#### シール保存制限（30枚）
- [ ] シールを30枚保存できることを確認
- [ ] 31枚目を保存しようとした時にペイウォールが表示されることを確認
- [ ] 複数シール一括保存で合計が30枚を超える場合にペイウォールが表示されることを確認
- [ ] ペイウォールの「あとで」を押した後、既存のシールが消えていないことを確認

#### ボード作成制限（1枚）
- [ ] デフォルトボード「はじめてのボード」が存在する状態で、新規ボード作成時にペイウォールが表示されることを確認
- [ ] HomeView の「新しいボードを作る」カードからペイウォールが出ることを確認
- [ ] HomeView の空状態の「+」ボタンからペイウォールが出ることを確認（ボードが1つ以上ある場合）
- [ ] BoardListView のツールバー「+」ボタンからペイウォールが出ることを確認

#### 枠線制限（中・太）
- [ ] 枠線設定で「なし」「細」が制限なく適用できることを確認
- [ ] 「中」「太」に PRO バッジが表示されていることを確認
- [ ] 「中」「太」を選択するとプレビューが表示されることを確認（プレビューは制限しない）
- [ ] 「中」「太」を選択した状態で「適用」を押すとペイウォールが表示されることを確認
- [ ] ペイウォールの「あとで」後、枠線が以前の設定のままであることを確認

#### 背景パターン制限（ストライプ・グラデーション）
- [ ] 無地・ドット・グリッドが制限なく使用できることを確認
- [ ] ストライプ・グラデーションに PRO バッジが表示されていることを確認
- [ ] ストライプ・グラデーションを選択するとプレビューが反映されることを確認
- [ ] ストライプ・グラデーションを選択した状態で「完了」を押すとペイウォールが表示されることを確認

#### 画像書き出し（ロゴ入り）
- [ ] ボードを画像保存した際、右下に「シールボード」ロゴが入っていることを確認
- [ ] ロゴが半透明で控えめに表示されていることを確認

### 2-2. ペイウォールUI確認

- [ ] ペイウォールが正しく表示されること（レイアウト崩れなし）
- [ ] 機能リスト5項目がすべて表示されていること
- [ ] 年額プランの価格（¥2,900/年）が正しく表示されること
- [ ] 月あたり価格と割引率が表示されること
- [ ] 「Proではじめる（年額）」ボタンが目立つこと
- [ ] 「月額プランで始める ¥380/月」テキストが表示されること
- [ ] 「購入を復元」リンクが表示されること
- [ ] 「あとで」で閉じられること

## 3. 購入フローのテスト

### 3-1. 年額プラン購入

1. ペイウォールを表示
2. 「Proではじめる（年額）」をタップ
3. StoreKit Testing の購入ダイアログが表示される
4. 「Confirm」をタップ
5. 確認事項:
   - [ ] 購入中のローディング表示が出ること
   - [ ] 購入成功後、ペイウォールが自動的に閉じること
   - [ ] 制限されていた機能が使えるようになること（PRO バッジが消えるなど）

### 3-2. 月額プラン購入

1. ペイウォールを表示
2. 「月額プランで始める」をタップ
3. 以降は年額と同じフロー

### 3-3. 購入キャンセル

1. ペイウォールを表示
2. CTAをタップ
3. StoreKit ダイアログで「Cancel」を選択
4. 確認事項:
   - [ ] ペイウォールが閉じないこと
   - [ ] エラー表示が出ないこと
   - [ ] 引き続き購入ボタンが有効であること

### 3-4. 購入復元

1. ペイウォールを表示
2. 「購入を復元」をタップ
3. 確認事項:
   - [ ] 既存の有効なサブスクリプションがあれば Pro 状態になること
   - [ ] 既存のサブスクリプションがなければ何も変わらないこと

## 4. エッジケーステスト

### 4-1. オフライン動作

1. 一度購入を完了させる
2. 機内モードにする
3. アプリを完全に終了して再起動
4. 確認事項:
   - [ ] Pro 状態が維持されていること（UserDefaults キャッシュ）
   - [ ] 制限なく機能が使えること

### 4-2. サブスクリプション期限切れ

Xcode で期限切れをシミュレート:
1. **Debug > StoreKit > Manage Transactions** を開く
2. 該当トランザクションを右クリック > **Refund** または **Expire**
3. アプリに戻る
4. 確認事項:
   - [ ] Pro 状態が解除されること
   - [ ] 制限が再度有効になること
   - [ ] 既に作成したボードやシールは削除されないこと

### 4-3. アプリ初回起動（購入なし）

1. シミュレータのデータを完全リセット
2. アプリを起動
3. 確認事項:
   - [ ] デフォルトの「はじめてのボード」が作成されること
   - [ ] 無料プランの制限が正しく動作すること
   - [ ] ペイウォールが適切なタイミングで表示されること

### 4-4. Pro購入後の状態持続

1. Pro を購入
2. アプリを完全終了
3. アプリを再起動
4. 確認事項:
   - [ ] Pro 状態が維持されていること
   - [ ] ボードを無制限に作成できること
   - [ ] すべての枠線・背景パターンが使えること

## 5. サンドボックステスト（実機）

### 5-1. App Store Connect 事前設定

本番リリース前に必要な作業:

1. **App Store Connect** でサブスクリプション商品を登録
   - サブスクリプショングループ: "StickerBoard Pro"
   - 月額: `com.solodev.StickerBoard.pro.monthly` ¥380
   - 年額: `com.solodev.StickerBoard.pro.yearly` ¥2,900
2. **サンドボックステスターアカウント** を作成
   - Settings > Users and Access > Sandbox Testers
3. 実機でサンドボックスアカウントでサインイン

### 5-2. サンドボックスの特徴

| 実環境 | サンドボックス |
|--------|-------------|
| 1ヶ月 | 5分 |
| 1年 | 1時間 |
| 自動更新 | 最大6回 |

### 5-3. 実機でのテスト手順

1. Xcode の Scheme で StoreKit Configuration を **なし** に設定
2. 実機にビルド・インストール
3. サンドボックスアカウントでサインイン
4. 購入フローを実施
5. 自動更新の確認（5分待機）

## 6. App Store 審査チェックリスト

### 6-1. 必須要件

- [ ] **購入復元ボタン**: ペイウォールに「購入を復元」が実装されていること
- [ ] **サブスクリプション説明**: 何に課金するのか明確に説明されていること
- [ ] **価格表示**: 実際の価格が `Product.displayPrice` から取得されていること（ハードコードNG）
- [ ] **利用規約リンク**: Apple 標準 EULA へのリンクがあること
- [ ] **プライバシーポリシーリンク**: リンクが設置されていること
- [ ] **無料で使える範囲**: 無料でもアプリの基本機能が使えること
- [ ] **強制課金なし**: 課金しなくてもアプリを使い続けられること

### 6-2. App Store Connect 設定

- [ ] サブスクリプショングループの作成
- [ ] 各商品の価格設定
- [ ] 商品の説明文（レビュー用）
- [ ] スクリーンショットにサブスクリプション画面を含める
- [ ] App Review Notes にテストアカウント情報を記載

### 6-3. よくあるリジェクト理由

| 理由 | 対策 |
|------|------|
| 復元ボタンがない | ペイウォールに「購入を復元」を設置済み |
| 価格がハードコード | `Product.displayPrice` を使用済み |
| サブスクの説明不足 | 機能リストを明示済み |
| 無料で何もできない | 切り抜き・フィルター・基本機能は無料 |
