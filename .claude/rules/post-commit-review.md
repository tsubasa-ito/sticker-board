# ルール: コミット後の公式ドキュメント準拠レビュー

## トリガー条件
git commit を実行した後、自動的に `/review-official-docs` コマンドを実行する。

## ルール
コードの変更をコミットした後は、必ず以下を実施すること：

1. `/review-official-docs` スキルを実行して、変更されたコードが各技術の公式ドキュメント・推奨パターンに準拠しているかレビューする
2. レビュー結果が **C（推奨パターンからの逸脱あり）** 以上の問題を含む場合、ユーザーに修正を提案する
3. レビュー結果が **D（重大な問題あり）** の場合、修正が必要であることを強調して報告する

## 対象
- `.swift` ファイルの変更を含むコミットのみ対象
- ドキュメントのみの変更（.md ファイルのみ）やプロジェクト設定ファイルのみの変更は対象外
