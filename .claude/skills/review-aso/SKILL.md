# ASOレビュー

`docs/appstore-metadata.md` の内容を読み込み、App Store Optimization のベストプラクティスに基づいてレビューを行い、改善提案を出力する。

## 手順

### 1. 現状データの取得

以下を読み込む：
- `docs/appstore-metadata.md`（メタデータ全文）
- `docs/PRODUCT_SPEC.md`（ターゲットユーザー・機能一覧の把握）

### 2. 各項目のレビュー

#### アプリ名（30文字以内）
- 主要キーワードが含まれているか
- ブランド名と機能訴求のバランスが取れているか
- 文字数が最大限活用されているか（30文字に近いほど有利）

#### サブタイトル（30文字以内）
- アプリ名で伝えられない追加価値を訴求できているか
- キーワードとして有効なワードが含まれているか
- アプリ名との重複を避けているか

#### キーワード（100文字以内）
- **文字数カウント**: 実際の文字数を数える（カンマ含む）
- **重複チェック**: アプリ名・サブタイトルに含まれるワードがキーワードにも入っていないか（重複すると100文字枠が無駄になる）
- **網羅性**: ターゲットユーザーが検索するワードが含まれているか
- **特化度**: 競合の多い汎用ワードより、ニッチで競合の少ないワードを優先しているか
- **スペース区切り活用**: App Store は各単語を個別にインデックスするため、スペース区切りで複数キーワードを組み合わせられているか
- **残り文字の活用**: 100文字に対して未使用の余白がないか

#### 説明文
- **冒頭フック（最重要）**: 「続きを読む」で隠れる前の最初の3〜4行で価値訴求できているか
- **感情訴求**: ターゲットユーザーの共感・欲求を刺激する書き出しになっているか
- **機能 vs ベネフィット**: 機能の列挙だけでなく「〜できる」「〜になれる」というベネフィット表現になっているか
- **ターゲット明示**: 誰向けのアプリかが伝わるか
- **Proプランの訴求位置**: 説明文の前半にProプランのメリットが触れられているか
- **CTA**: ダウンロードを促す行動喚起があるか

#### プロモーションテキスト（170文字以内）
- 季節・イベント・新機能に合わせた時事性があるか
- アプリの核心的な価値を一文で伝えられているか
- 文字数が最大限活用されているか
- **審査不要のため即変更可能**（CVR改善の最速レバー）

#### What's New
- ユーザー目線のメリットで書かれているか（機能名の羅列ではなく「〜できるようになりました」形式）
- 新機能・改善・修正が適切に分類されているか

#### スクリーンショット（CVRへの影響が最大）
- `docs/appstore-metadata.md` のキャプション案を確認する
- 最初の1〜2枚でアプリの核心価値が伝わるか
- テキストキャプションがシンプルで読みやすいか
- iPhone 6.9インチ・iPad 13インチ両方に対応しているか
- 枚数は5〜8枚が推奨（現状の枚数を確認する）

### 3. Analytics観点のチェック

以下の指標が把握できている場合、対応する改善ポイントを特定する：

| 指標 | 業界平均 | 改善余地があるレバー |
|------|---------|-------------------|
| CVR（コンバージョン率） | 3〜5% | スクリーンショット、アプリ名、サブタイトル |
| インプレッション → プロダクトページ閲覧 | — | キーワード最適化 |
| プロダクトページ閲覧 → ダウンロード | 60〜80% | スクリーンショット、説明文冒頭、評価・レビュー |
| 評価件数 | アプリ規模による | レビュー訴求タイミングの最適化 |

### 4. レポート出力

以下のフォーマットで出力する：

```
## ASOレビュー結果

### 総合評価: [A / B / C / D]
- A: ベストプラクティスに完全準拠、即座の改善余地なし
- B: 軽微な改善点あり（更なる最適化で上積み可能）
- C: 改善余地あり（実施すれば CVR / インプレッション向上が見込める）
- D: 重大な機会損失あり（早急に対処すべき項目がある）

---

### 検出事項

#### [項目名] - [重要度: 高/中/低]
- **現状**: （実際の設定内容）
- **問題点**: （何が課題か）
- **改善案**: （具体的な修正案・代替案）

---

### 良好な点
- （すでに最適化できている項目）

---

### 優先対応リスト
1. 【高】...
2. 【中】...
3. 【低】...
```

### 5. 注意事項
- `docs/appstore-metadata.md` の内容が App Store Connect と同期されていることが前提。乖離が疑われる場合はその旨を冒頭に記載する
- 審査中バージョンがある場合、説明文・スクリーンショット等の変更は次回審査提出時まで反映されないことを考慮する
- **プロモーションテキストは審査不要でいつでも変更可能**（即効性が高い最速レバー）
- **キーワードはバージョン提出時にのみ変更可能**（審査が必要）
