# 継続的ASO最適化タスク - シールボード

**作成日:** 2026-04-09
**対象期間:** 2026年4月以降（継続的）

---

## 日次タスク（15分/日）

### レビュー対応

- [ ] App Store Connect > レビュー を確認する
- [ ] 新しいレビューがあれば review-responses.md のテンプレートを使って24時間以内に返信する
- [ ] ネガティブレビュー（星1-2）は即日返信する

### クラッシュ監視

- [ ] Firebase Crashlytics ダッシュボードを確認する
- [ ] 新しいクラッシュが増加していないか確認する
- [ ] クリティカルなクラッシュ（影響ユーザー10人以上）がある場合は即対応

---

## 週次タスク（1時間/週、毎週月曜推奨）

### キーワード順位確認

- [ ] Safariプライベートモードで以下のキーワードを検索し、シールボードの順位を記録する:
  1. 「シール帳」
  2. 「デジタルシール帳」
  3. 「推し活 シール」
  4. 「シール コレクション」
  5. 「写真 切り抜き」
- [ ] 前週との比較を記録する
- [ ] 順位が大きく変動した場合は原因を分析する

### App Analytics 確認

- [ ] App Store Connect > App Analytics を開く
- [ ] インプレッション数を確認する（前週比）
- [ ] プロダクトページビュー数を確認する
- [ ] ダウンロード数を確認する（前週比）
- [ ] コンバージョン率を確認する（インプレッション → ダウンロード）
- [ ] ソース別（検索、ブラウズ、紹介）の割合を確認する

### 競合モニタリング

- [ ] 以下の直接競合のApp Storeページを確認する:
  1. デジタルシール帳
  2. ぺたこれ
  3. ぺた帳
- [ ] スクリーンショットやメタデータに変更がないか確認する
- [ ] 新しい競合アプリが登場していないか「シール帳」で検索して確認する

## 月次タスク（2時間/月、毎月第1月曜推奨）

### ASO健全性チェック

- [ ] キーワード順位の月次推移をグラフ化する
- [ ] ダウンロード数の月次推移を確認する
- [ ] コンバージョン率の月次推移を確認する
- [ ] 目標に対する進捗を確認する

### プロモーションテキスト更新

- [ ] 季節やイベントに合わせたプロモーションテキストを検討する
- [ ] apple-metadata.md のテンプレートから選択または新規作成する
- [ ] App Store Connect で更新する（審査不要で即反映）

| 時期 | テーマ | 例 |
|------|-------|---|
| 4月 | 新生活・入学 | 「新しいシール帳、始めませんか?」 |
| 5月 | GW・母の日 | 「GWの思い出をシールに!」 |
| 7-8月 | 夏休み・イベント | 「夏フェス・イベントの限定ステッカーを永久保存!」 |
| 10月 | ハロウィン | 「ハロウィン限定シールを集めよう」 |
| 12月 | クリスマス・年末 | 「今年の推し活コレクション、ボードに残そう」 |
| 通年 | 推し活シーズン | ライブ・イベント前後にプロモーション |

### レビュー分析

- [ ] 直近1ヶ月のレビューを全件確認する
- [ ] 共通する要望・不満を抽出する
- [ ] 機能開発の優先順位に反映する

### 競合詳細分析

- [ ] 直接競合3社のレビュー内容を確認する
- [ ] 競合の新機能・更新内容を確認する
- [ ] 新たな差別化ポイントがないか検討する

## 四半期タスク（半日/四半期）

### キーワードフィールドの見直し

- [ ] 過去3ヶ月のキーワード順位データを分析する
- [ ] 効果の低いキーワードを特定する
- [ ] トレンドキーワードを調査する
- [ ] キーワードフィールドを更新する（次回バージョン提出時）

### スクリーンショットの更新

- [ ] A/Bテストの結果に基づいてスクリーンショットを最適化する
- [ ] 新機能が追加された場合はスクリーンショットに反映する
- [ ] 競合のスクリーンショットデザインとの差別化を確認する

### 説明文の更新

- [ ] 新機能や改善点を説明文に反映する
- [ ] ユーザーレビューで多い質問への回答を説明文に追加する
