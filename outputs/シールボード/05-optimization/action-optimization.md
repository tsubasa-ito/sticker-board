# 継続的最適化フェーズ アクションチェックリスト

**フェーズ:** 05-optimization
**作成日:** 2026-04-09
**対象:** ASO最適化適用後の継続的な改善活動

---

## 初期セットアップ（1回のみ）

- [ ] ongoing-tasks.md の日次・週次・月次タスクを確認する
- [ ] カレンダーにリマインダーを設定する:
  - 毎日: レビュー確認（5分）
  - 毎週月曜: キーワード順位 + Analytics確認（30分）
  - 毎月第1月曜: 月次レビュー（2時間）
- [ ] キーワード順位トラッキング用のスプレッドシートを作成する
- [ ] review-responses.md のテンプレートを確認・カスタマイズする

## レビュー獲得施策

### アプリ内レビューリクエストの最適化

現在の ReviewRequestManager の設定:
- トリガー: シール5/15/30枚目、ボード新規作成時、起動5回目
- クールダウン: 90日
- 年間上限: 3回

- [ ] 上記の条件が適切か検討する
- [ ] 特に「シール5枚目」のタイミングが早すぎないか確認する
- [ ] 起動5回目のトリガーが有効に機能しているか確認する

### SNSでのレビュー依頼（外部施策）

- [ ] Twitter/Xでアプリの使い方を投稿し、レビューを依頼する
- [ ] アプリに関するハッシュタグを決める（例: #シールボード #推し活シール帳）
- [ ] ユーザーが作成したボードのシェア投稿をリツイート/引用する

## コンバージョン率改善

- [ ] スクリーンショットの効果を A/B テストで検証する（03-testing参照）
- [ ] 説明文の最初の3行を定期的に見直す（検索結果で表示される部分）
- [ ] プロモーションテキストを季節イベントに合わせて更新する

## 検索順位改善

- [ ] 週次のキーワード順位データから、上昇・下降トレンドを分析する
- [ ] 上昇中のキーワードはそのまま維持する
- [ ] 効果のないキーワードは四半期ごとに入れ替える
- [ ] 競合が使い始めた新しいキーワードを把握する

---

## 検証基準

| 項目 | 基準 |
|------|------|
| 日次タスク | レビュー確認が毎日実施されている |
| 週次タスク | キーワード順位とAnalyticsが毎週記録されている |
| 月次タスク | プロモーションテキストが季節に合わせて更新されている |
| KPI進捗 | ongoing-tasks.md の目標に対して順調に進捗している |

ongoing-tasks.md のスケジュールに従って、日次・週次・月次・四半期のタスクを実行してください。
