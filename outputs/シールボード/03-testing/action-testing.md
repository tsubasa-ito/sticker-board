## 前提タスク（テスト開始前に必須）

- [ ] スクリーンショットが5枚以上 App Store Connect に登録されていること
- [ ] バージョン1.3.0が App Store で公開されていること
- [ ] 日本語ローカライゼーションが設定されていること
- [ ] App Analytics でインプレッションデータが取得できる状態であること

---

## テスト1: スクリーンショット1枚目のバリエーション

### 準備

- [ ] コントロール用スクリーンショット1枚目（完成ボード）を確定する
- [ ] トリートメントA用（AI切り抜きビフォーアフター）を作成する
- [ ] トリートメントB用（推し活ボード完成例）を作成する
- [ ] 3つのバリエーションのサイズが同一であることを確認する

### 設定

- [ ] App Store Connect > Product Page Optimization を開く
- [ ] 「テストを作成」をクリックする
- [ ] テスト名を入力: 「SS1枚目_ヒーロー vs AI vs 推し活」
- [ ] ローカライゼーション: 日本語 を選択する
- [ ] トリートメントを2つ追加する
- [ ] 各トリートメントにスクリーンショットをアップロードする
- [ ] トラフィック配分を設定: コントロール34% / A: 33% / B: 33%
- [ ] テスト開始日を設定: 2026-04-21（推奨）
- [ ] テストを開始する

### モニタリング

- [ ] テスト開始後3日目: インプレッション数を確認（十分なサンプルがあるか）
- [ ] テスト開始後7日目: 中間結果を確認
- [ ] テスト開始後14日目: 結果を確認
- [ ] 統計的有意性に達していない場合: テスト期間を延長（最大28日）
- [ ] 統計的有意性に達した場合: 勝者を採用

### 結果記録

- [ ] テスト結果をスプレッドシートに記録する:
  - テスト名、期間、コントロールのコンバージョン率、各トリートメントのコンバージョン率、勝者
- [ ] 勝者バリエーションを「適用」する
